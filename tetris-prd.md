# PRD: 테트리스 게임 (1인용 / 2인용)

- **문서 버전**: 1.0
- **작성일**: 2026-07-12
- **개발 환경**: Antigravity + Claude Code
- **대상 플랫폼**: 웹 브라우저 (데스크톱 우선)

---

## 1. 개요 (Overview)

키보드로 플레이하는 웹 기반 테트리스 게임을 개발한다. 1인용 클래식 모드와, 한 대의 PC에서 두 명이 같은 키보드로 대결하는 로컬 2인용 배틀 모드를 제공한다.

> **가정**: 2인용은 온라인 멀티가 아닌 **로컬(같은 키보드) 대전**으로 한다. 온라인 대전은 v2 이후 범위로 분리한다.

### 1.1 목표 (Goals)

- 표준 테트리스 가이드라인(SRS 회전, 7-bag 랜덤)을 따르는 정확한 게임플레이 구현
- 별도 설치 없이 브라우저에서 즉시 플레이 가능
- 2인 대전 시 가비지 라인 공방을 통한 경쟁 재미 제공
- 60fps의 부드러운 렌더링과 즉각적인 입력 반응성

### 1.2 비목표 (Non-Goals)

- 온라인 멀티플레이 (v2)
- 모바일 터치 조작 (v2)
- 계정/로그인, 서버 저장 랭킹 (로컬 저장만 지원)
- 사운드 트랙 라이선스 음원 (무료/자체 제작 효과음만 사용)

| 친구/가족 2인 | 한 키보드로 나란히 앉아 대전, 3판 2선승 등 승부 |

## 3. 게임 모드

### 3.1 1인용 모드 (Marathon)

- 무한 진행, 레벨이 오를수록 낙하 속도 증가
- 10라인 클리어마다 레벨 +1 (레벨 1 시작, 최대 레벨 20에서 속도 고정)
- 게임 오버 조건: 새 블록이 스폰 위치에 놓일 수 없을 때
- 게임 오버 시 점수/라인/레벨 표시, 로컬 최고 기록(localStorage) 갱신

### 3.2 2인용 배틀 모드 (Local Versus)

- 화면을 좌/우로 분할하여 두 개의 보드를 동시 표시
- 라인 클리어 시 상대에게 가비지 라인 전송 (아래 공격 테이블 참조)
- 승리 조건: 상대가 먼저 게임 오버(탑아웃)되면 승리
- 라운드제: 기본 3판 2선승 (설정에서 1판, 5판 3선승 선택 가능)
- 두 플레이어의 낙하 속도는 동일하게 고정(레벨 5 상당), 시간 경과에 따라 5분마다 한 단계씩 상승

#### 공격 테이블 (가비지 라인 수)

| 클리어 종류 | 전송 라인 |
|---|---|
| 싱글 (1줄) | 0 |
| 더블 (2줄) | 1 |
| 트리플 (3줄) | 2 |
| 테트리스 (4줄) | 4 |
| T-스핀 싱글 | 2 |
| T-스핀 더블 | 4 |
| T-스핀 트리플 | 6 |
| 백투백 보너스 | +1 |
| 콤보 | 콤보 수에 따라 +1 ~ +4 |

#### 가비지 규칙

- 가비지 라인은 무작위 한 칸이 비어 있는 회색 줄로 보드 하단에서 올라옴
- **상쇄(Counter)**: 자신에게 대기 중인 가비지가 있을 때 라인을 클리어하면 그만큼 차감 후 잔여분만 전송
- 가비지는 즉시 적용되지 않고 대기열에 쌓였다가, 다음 블록 고정(lock) 시점에 올라옴
- 대기 가비지 양은 보드 옆 게이지로 시각화

## 4. 핵심 게임 메커니즘 (공통)

### 4.1 보드 및 블록

- 보드 크기: 10 x 20 (상단에 보이지 않는 버퍼 2줄 추가, 총 10 x 22)
- 7종 테트로미노: I, O, T, S, Z, J, L — 표준 색상 사용
  - I: 하늘색, O: 노랑, T: 보라, S: 초록, Z: 빨강, J: 파랑, L: 주황

### 4.2 랜덤 시스템

- **7-bag 랜덤라이저**: 7종 블록을 한 가방에 넣고 섞어 순서대로 지급, 가방이 비면 새로 섞음

### 4.3 회전 시스템

- **SRS (Super Rotation System)** 적용, 벽킥(wall kick) 테이블 포함
- T-스핀 판정: T 블록이 회전으로 고정되고, 네 모서리 중 3개 이상이 채워져 있으면 T-스핀

### 4.4 조작 메커니즘

- **홀드(Hold)**: 현재 블록을 1회 보관/교체 (블록 고정 전까지 재사용 불가)
- **고스트 피스**: 하드드롭 착지 지점을 반투명하게 표시
- **소프트 드롭**: 누르는 동안 빠르게 하강 (기본 중력의 20배)
- **하드 드롭**: 즉시 바닥으로 낙하 후 고정
- **락 딜레이**: 착지 후 0.5초 유예, 이동/회전 시 리셋 (최대 15회 리셋 후 강제 고정)
- **DAS/ARR**: 키를 누르고 있을 때 자동 반복 이동 (DAS 167ms, ARR 33ms 기본값, 설정에서 조정 가능)

### 4.5 점수 시스템 (1인용)

| 액션 | 점수 |
|---|---|
| 싱글 | 100 × 레벨 |
| 더블 | 300 × 레벨 |
| 트리플 | 500 × 레벨 |
| 테트리스 | 800 × 레벨 |
| T-스핀 싱글 | 800 × 레벨 |
| T-스핀 더블 | 1200 × 레벨 |
| T-스핀 트리플 | 1600 × 레벨 |
| 소프트 드롭 | 1점/칸 |
| 하드 드롭 | 2점/칸 |
| 백투백 | 해당 점수 × 1.5 |
| 콤보 | 50 × 콤보 수 × 레벨 |

### 4.6 낙하 속도 (중력)

- 레벨 1: 1초/칸에서 시작, 레벨이 오를수록 표준 가이드라인 곡선에 따라 가속
- 레벨 20: 약 0.02초/칸 (사실상 즉시 낙하)

## 5. 조작 체계

### 5.1 1인용 키 배치

| 동작 | 키 |
|---|---|
| 좌/우 이동 | ← / → |
| 소프트 드롭 | ↓ |
| 하드 드롭 | Space |
| 시계 회전 | ↑ 또는 X |
| 반시계 회전 | Z |
| 홀드 | C 또는 Shift |
| 일시정지 | Esc 또는 P |

### 5.2 2인용 키 배치 (한 키보드)

| 동작 | Player 1 (왼쪽) | Player 2 (오른쪽) |
|---|---|---|
| 좌/우 이동 | A / D | ← / → |
| 소프트 드롭 | S | ↓ |
| 하드 드롭 | W | ↑ |
| 시계 회전 | F | . (마침표) |
| 반시계 회전 | G | , (쉼표) |
| 홀드 | R 또는 Q | / (슬래시) |
| 일시정지 | Esc (공통) | Esc (공통) |

- 키 동시 입력 충돌(키보드 고스팅)을 고려해 서로 다른 키 영역에 배치
- 설정 화면에서 키 리매핑 지원 (v1.1 목표, v1은 고정 키)

## 6. UI/UX 요구사항

### 6.1 화면 구성

1. **메인 메뉴**: 1인용 시작 / 2인용 시작 / 기록 / 조작법 (설정 화면은 v1.1)
   - **기록 화면**: 하이스코어 톱10, 최근 기록, TXT 내보내기/불러오기/파일 연결 버튼
2. **1인용 게임 화면**: 중앙 보드, 우측에 다음 블록 5개 미리보기·점수·레벨·라인, 좌측에 홀드 블록
3. **2인용 게임 화면**: 좌우 대칭 2개 보드, 각 보드마다 홀드·넥스트(3개)·가비지 게이지·승수 표시
4. **일시정지 오버레이**: 계속하기 / 재시작 / 메뉴로
5. **결과 화면**: 1인용은 점수·기록, 2인용은 라운드 승자·시리즈 스코어와 다음 라운드/재대결 버튼

### 6.2 시각 효과

- 라인 클리어 시 플래시/파티클 애니메이션 (150ms 이내, 게임 흐름 방해 금지)
- 테트리스/T-스핀/백투백 시 텍스트 팝업 ("TETRIS!", "T-SPIN DOUBLE!" 등)
- 하드 드롭 시 짧은 화면 진동 효과
- 다크 테마 기본, 보드 격자선 은은하게 표시

### 6.3 사운드 (선택 구현)

- 효과음: 이동, 회전, 고정, 라인 클리어, 테트리스, 가비지 수신, 게임 오버
- 음소거 토글 제공, 설정은 localStorage에 저장

### 6.4 반응형

- 데스크톱 우선. 최소 1280px 너비 기준으로 2인용 화면이 잘리지 않아야 함
- 창 크기에 따라 보드 스케일 자동 조정

## 7. 기술 요구사항

### 7.1 기술 스택 (Claude Code 구현 권장안)

- **언어**: TypeScript
- **렌더링**: HTML5 Canvas (게임 보드) + DOM (메뉴/HUD)
- **빌드**: Vite
- **상태 관리**: 프레임워크 없이 게임 루프 기반 순수 구현 (React 등 불필요)
- **저장**: localStorage (최고 기록, 설정) + `scores.txt` 파일 동기화 — File System Access API(Chrome/Edge) 자동 기록, 기타 브라우저는 내보내기/불러오기 폴백 (상세: implementation_plan.md 0장)
- **테스트**: Vitest — 회전/벽킥/라인 클리어/가비지 로직 단위 테스트

### 7.2 아키텍처 원칙

- 게임 로직(코어)과 렌더링/입력을 분리하여 코어는 순수 함수로 테스트 가능하게 구현
- 2인용은 동일한 게임 코어 인스턴스 2개를 생성하고, 가비지 교환은 이벤트 버스로 연결
- 게임 루프: `requestAnimationFrame` 기반 고정 타임스텝 (60 ticks/sec), 렌더링과 로직 틱 분리
- 입력은 keydown/keyup 상태 폴링 방식 (DAS/ARR 구현을 위해 필수)

### 7.3 성능

- 2인용 동시 플레이 시에도 60fps 유지
- 입력 지연 1프레임(≈16ms) 이내
- 초기 로딩 3초 이내 (에셋 최소화)

### 7.4 프로젝트 구조 (권장)

```
src/
  core/          # 게임 로직 (보드, 피스, 회전, 랜덤, 점수, 가비지)
  input/         # 키 입력, DAS/ARR 처리
  render/        # Canvas 렌더러
  ui/            # 메뉴, HUD, 결과 화면
  audio/         # 효과음 (선택)
  main.ts        # 진입점, 게임 루프
tests/
  core/          # 코어 로직 단위 테스트
```

## 8. 개발 마일스톤 (Claude Code 작업 단위)

| 단계 | 내용 | 완료 기준 |
|---|---|---|
| M1 | 프로젝트 셋업 + 코어 로직 (보드, 7-bag, SRS, 라인 클리어) | 코어 단위 테스트 통과 |
| M2 | 1인용 플레이 가능 버전 (렌더링, 입력, 중력, 점수, 게임 오버) | 브라우저에서 1인 마라톤 플레이 가능 |
| M3 | 홀드, 고스트, 하드드롭, 락 딜레이, DAS/ARR, T-스핀 판정 | 표준 조작감 완성 |
| M4 | 2인용 분할 화면 + 가비지 공방 + 라운드제 | 2인 대전 한 세트 완주 가능 |
| M5 | 메뉴/결과 화면, 이펙트, 최고 기록 저장, 폴리싱 | 전체 플로우 완성 |
| M6 (선택) | 사운드, 키 리매핑, 설정 화면 | v1.1 |

## 9. 수용 기준 (Acceptance Criteria)

- [ ] 7-bag: 어떤 시점에서도 같은 블록이 연속 지급되는 간격이 최대 12개를 넘지 않음
- [ ] SRS: 표준 벽킥 테이블의 모든 킥 케이스가 단위 테스트로 검증됨
- [ ] T-스핀 더블 수행 시 점수와 공격 라인이 정확히 계산됨
- [ ] 2인용에서 P1이 테트리스 클리어 시 P2 보드에 가비지 4줄이 다음 고정 시점에 올라옴
- [ ] 대기 가비지가 있는 상태에서 라인 클리어 시 상쇄가 정확히 동작함
- [ ] 두 플레이어가 동시에 키를 입력해도 조작 누락이 없음
- [ ] 게임 오버/라운드 종료/시리즈 종료 플로우가 끊김 없이 이어짐
- [ ] 새로고침 후에도 1인용 최고 기록이 유지됨
